- One row per spec under [00-roadmap.md](./00-roadmap.md).

- Date format: `YYYY-MM-DD`. Always absolute, never relative.
- When marking `complete`, also add a Status header inside the spec file itself.
- When state changes, append a one-line entry to the changelog at the bottom (newest first).
